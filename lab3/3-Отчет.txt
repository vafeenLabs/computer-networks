1. Создание базовой топологии сети
1.1. В Cisco Packet Tracer разместить устройства согласно заданной схеме: маршрутизаторы (Router), коммутаторы (Switch), рабочие станции (PC) и серверы (Server).

1.2. Соединить устройства:

Использовать кабели Copper Straight-Through для подключения конечных устройств (ПК и серверов) к коммутаторам.
Применить Copper Cross-Over для соединения маршрутизаторов между собой.
1.3. Убедиться, что сеть соответствует заданной топологии.

2. Настройка IP-адресов
2.1. Разделить сеть на подсети в соответствии с топологией.

2.2. Задать статические IP-адреса для ПК и серверов:

Адреса устройств должны принадлежать их подсетям (например, 192.168.0.x, 192.168.1.x и т.д.).
Для каждого ПК указать IP-адрес, маску подсети и шлюз:
Пример:
IP Address: 192.168.0.10
Subnet Mask: 255.255.255.0
Default Gateway: 192.168.0.1
2.3. Настроить интерфейсы маршрутизаторов:

Присвоить IP-адреса интерфейсам, подключённым к соответствующим подсетям, через командную строку маршрутизатора:

Router> enable  
Router# configure terminal  
Router(config)# interface FastEthernet0/0  
Router(config-if)# ip address 192.168.0.1 255.255.255.0  
Router(config-if)# no shutdown  
3. Настройка динамической маршрутизации (RIP)
3.1. Активировать протокол RIP на всех маршрутизаторах:

Router> enable  
Router# configure terminal  
Router(config)# router rip  
Router(config-router)# version 2  
Router(config-router)# network 192.168.0.0  
Router(config-router)# network 10.0.0.0  
Router(config-router)# network 11.0.0.0  

3.2. Убедиться, что маршрутизаторы обмениваются маршрутами корректно.

4. Конфигурация сервисов (DNS, Email, HTTP)
4.1. Выбрать серверы для каждого из сервисов.

4.2. Настроить службы:

DNS-сервер:
Задать доменные имена (например, lab3.ru) и их IP-адреса.
Email-сервер:
Включить SMTP и POP3.
Создать учётные записи пользователей.
WEB-сервер:
Включить HTTP.
Настроить основной веб-сайт.
5. Тестирование сети
5.1. Проверить связь между устройствами:

Использовать ping для отправки пакетов между устройствами внутри одной подсети и между разными подсетями.
5.2. Убедиться в работоспособности сервисов:

Проверить доступность веб-сайта через DNS, отправив запрос по доменному имени.
Проверить возможность отправки и получения писем через Email-сервер.
5.3. Проанализировать прохождение пакетов:

Убедиться, что маршрутизаторы корректно перенаправляют пакеты между подсетями.
Проверить, как пакеты проходят через DNS, Email и HTTP-сервисы.
