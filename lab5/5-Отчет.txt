Предусмотрел участок сети, имеющий альтернативный маршрут

Допоплнил сеть частной подсетью, в которой предусмотрел 5 рабочих станций, с настроенным DNS, EMAIL и HTTP серверами.

Провёл настройку сети

При помощи протокола RIP получил доступ из каждой подсети в кажду путём
прописывания всех подсетей в нужном окне

Для того что бы соеденить роутеры при помощи протокола rip, нужно в каждом
роутере в прописать все соседние подсети.

Для того что бы посмотреть, что роутеры соеденины именно при помощи Rip протокола,
нужно открыть таблицу маршрутизации
R в столбце type означает rip, можно обратить внимание на метрику, она отличается от connected и static.

Альтернативные маршруты тоже работают. Пример:
Для попадания из сети 192.168.1.0 в 192.168.2.0 пакет пойдёт через сеть 192.168.12.0

Если оторвать этот провод, то пакет полетит сначала через 192.168.13.0, а затем
в 192.168.23.0, после чего попадёт в нужную сеть

Организовал демилитаризованную зону с использованием списков достоупа:
en
conf t
ip access-list extended название
	permit icmp подсеть обратная_маска any //Позволяет пинговать с конкретной сети, запрещая все остальные
	permit udp any any eq domain
	permit udp any eq domain any //Позволяет работать с DNS сервером
	permit tcp any any eq www //Позволяет работать с WEB сервером
	permit tcp any any eq smtp
	permit tcp any any eq pop3 //Позволяет работать с EMAIL сервером

У нас будут работать все службы, но при этом ping будет только из указанной сети
